                            manager01 / admin123
                            sale01 / admin123
                            tech01 / admin123

💡 MẸO:
- Sau khi login, copy "accessToken" vào tab Authorization → Type: Bearer Token
- Refresh token tự động được set trong Cookie (HttpOnly)
- Pageable: thêm ?page=0&size=10&sort=createdAt,desc vào GET list
- Các ID trong body JSON (customerId, productId,...) phải thay bằng ID thật trong DB

================================================================================
  LUỒNG 1: XÁC THỰC (AUTH) — Không cần Token
================================================================================

────────────────────────────────────────────────────────────────────────────────
1.1  ĐĂNG KÝ KHÁCH HÀNG
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/auth/register
Content-Type: application/json

{
    "username": "khachhang01",
    "password": "123456",
    "fullName": "Nguyễn Văn Khách",
    "phone": "0901234567",
    "address": "123 Đường ABC, Quận 1, TP.HCM"
}

→ 201: Đăng ký thành công, trả về accessToken (tự động đăng nhập)
→ 400: Username hoặc phone đã tồn tại

────────────────────────────────────────────────────────────────────────────────
1.2  ĐĂNG NHẬP (ADMIN)
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/auth/login
Content-Type: application/json

{
    "username": "admin",
    "password": "admin123"
}

→ 200: Trả về accessToken + cookie refreshToken
→ 401: Sai tên đăng nhập hoặc mật khẩu
→ 403: Tài khoản bị khóa

────────────────────────────────────────────────────────────────────────────────
1.3  ĐĂNG NHẬP (SALE)
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/auth/login
Content-Type: application/json

{
    "username": "sale01",
    "password": "admin123"
}

────────────────────────────────────────────────────────────────────────────────
1.4  ĐĂNG NHẬP (TECHNICIAN)
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/auth/login
Content-Type: application/json

{
    "username": "tech01",
    "password": "admin123"
}

────────────────────────────────────────────────────────────────────────────────
1.5  ĐĂNG NHẬP (CUSTOMER — sau khi đã đăng ký)
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/auth/login
Content-Type: application/json

{
    "username": "khachhang01",
    "password": "123456"
}

────────────────────────────────────────────────────────────────────────────────
1.6  LÀM MỚI TOKEN
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/auth/refresh

(Không cần body — Postman tự gửi cookie refreshToken)
→ 200: Trả về accessToken mới + cookie refreshToken mới
→ 401: Refresh token không hợp lệ hoặc hết hạn

────────────────────────────────────────────────────────────────────────────────
1.7  ĐĂNG XUẤT
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/auth/logout

(Không cần body — server xóa refresh token trong DB + clear cookie)
→ 200: Đăng xuất thành công

────────────────────────────────────────────────────────────────────────────────
7.2  CHI TIẾT ĐƠN HÀNG — Token: bất kỳ role
────────────────────────────────────────────────────────────────────────────────
GET http://localhost:8080/api/v1/orders/1
Authorization: Bearer {{accessToken}}

────────────────────────────────────────────────────────────────────────────────
7.3  TẠO ĐƠN HÀNG — Token: bất kỳ role (bao gồm CUSTOMER)
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/orders
Authorization: Bearer {{accessToken}}
Content-Type: application/json

{
    "customerId": 1,
    "shippingAddress": "123 Đường ABC, Quận 1, TP.HCM",
    "items": [
        {
            "productId": 1,
            "quantity": 2
        },
        {
            "productId": 2,
            "quantity": 5
        }
    ]
}

⚠️ Thay customerId và productId bằng ID thật trong DB!
⚠️ Sản phẩm phải có đủ tồn kho (stockQuantity >= quantity)

→ 201: Tạo thành công (status = PENDING, tự sinh orderCode)
→ 400: Không đủ tồn kho
→ 404: Customer hoặc Product không tồn tại

────────────────────────────────────────────────────────────────────────────────
7.4  CẬP NHẬT TRẠNG THÁI: PENDING → PROCESSING — Token: ADMIN, MANAGEMENT
────────────────────────────────────────────────────────────────────────────────
PATCH http://localhost:8080/api/v1/orders/1/status
Authorization: Bearer {{accessToken_admin}}
Content-Type: application/json

{
    "status": "PROCESSING"
}

────────────────────────────────────────────────────────────────────────────────
7.5  CẬP NHẬT TRẠNG THÁI: PROCESSING → COMPLETED — Token: ADMIN, MANAGEMENT
────────────────────────────────────────────────────────────────────────────────
PATCH http://localhost:8080/api/v1/orders/1/status
Authorization: Bearer {{accessToken_admin}}
Content-Type: application/json

{
    "status": "COMPLETED"
}

⚠️ Khi COMPLETED: tự động trừ stockQuantity các sản phẩm trong đơn
→ 400: Nếu stockQuantity không đủ → rollback toàn bộ

────────────────────────────────────────────────────────────────────────────────
7.6  CẬP NHẬT TRẠNG THÁI: COMPLETED → CANCELLED (hoàn đơn) — Token: ADMIN
────────────────────────────────────────────────────────────────────────────────
PATCH http://localhost:8080/api/v1/orders/1/status
Authorization: Bearer {{accessToken_admin}}
Content-Type: application/json

{
    "status": "CANCELLED"
}

⚠️ Khi hủy đơn đã COMPLETED: tự động cộng lại stockQuantity

────────────────────────────────────────────────────────────────────────────────
  LUỒNG TRẠNG THÁI ĐƠN HÀNG:
  PENDING → PROCESSING → COMPLETED → CANCELLED (hoàn đơn)
  PENDING → CANCELLED (hủy trước khi xử lý)
  PROCESSING → CANCELLED (hủy đang xử lý)

────────────────────────────────────────────────────────────────────────────────
GET http://localhost:8080/api/v1/maintenance-bookings/1

────────────────────────────────────────────────────────────────────────────────
8.3  TẠO LỊCH BẢO TRÌ — Token: bất kỳ role (bao gồm CUSTOMER)
────────────────────────────────────────────────────────────────────────────────
POST http://localhost:8080/api/v1/maintenance-bookings
Authorization: Bearer {{accessToken}}
Content-Type: application/json

{
    "customerId": 1,
    "serviceId": 1,
    "bookingDate": "2026-03-15T09:00:00",
    "notes": "Máy lọc nước chảy yếu, cần kiểm tra"
}

⚠️ bookingDate phải là ngày trong tương lai!
⚠️ Thay customerId và serviceId bằng ID thật trong DB

→ 201: Tạo thành công (status = PENDING)
→ 404: Customer hoặc Service không tồn tại

────────────────────────────────────────────────────────────────────────────────
8.4  GÁN KỸ THUẬT VIÊN (PENDING → CONFIRMED) — Token: ADMIN hoặc MANAGEMENT
────────────────────────────────────────────────────────────────────────────────
PATCH http://localhost:8080/api/v1/maintenance-bookings/1/assign
Authorization: Bearer {{accessToken_admin}}
Content-Type: application/json

{
    "technicianId": 4
}

⚠️ technicianId là ID của user có role TECHNICIAN (VD: tech01 = ID 4)
→ 200: Gán thành công (status chuyển sang CONFIRMED)
→ 400: Lịch không ở trạng thái PENDING

────────────────────────────────────────────────────────────────────────────────
8.5  HOÀN THÀNH (CONFIRMED → COMPLETED) — Token: ADMIN hoặc TECHNICIAN (được gán)
────────────────────────────────────────────────────────────────────────────────
PATCH http://localhost:8080/api/v1/maintenance-bookings/1/complete
Authorization: Bearer {{accessToken_tech}}
Content-Type: application/json

{
    "notes": "Đã thay 3 lõi lọc số 1-2-3, vệ sinh màng RO. Máy hoạt động tốt."
}

(body không bắt buộc — nếu không gửi body cũng OK)

→ 200: Hoàn thành (status chuyển sang COMPLETED)
→ 400: Lịch không ở trạng thái CONFIRMED
→ 403: Không phải kỹ thuật viên được gán
→ 409: Lịch đã hoàn thành rồi

────────────────────────────────────────────────────────────────────────────────
8.6  HỦY LỊCH (→ CANCELLED) — Token: ADMIN hoặc MANAGEMENT
────────────────────────────────────────────────────────────────────────────────
PATCH http://localhost:8080/api/v1/maintenance-bookings/1/cancel
Authorization: Bearer {{accessToken_admin}}

(Không cần body)

→ 200: Đã hủy
→ 409: Lịch đã hoàn thành, không thể hủy

────────────────────────────────────────────────────────────────────────────────
  LUỒNG TRẠNG THÁI LỊCH BẢO TRÌ:
  PENDING → CONFIRMED → COMPLETED
  PENDING → CANCELLED
  CONFIRMED → CANCELLED
